前言

之前我在github上搭建了自己的博客项目，通过github page成功上线。当时我的域名就是gethub page的默认域名：

在国内访问加载速度有些慢，于是我找了找网上的教程学习该如何加速访问，提高速度。

本期内容

✅ 使用腾讯云注册域名并在github上配置
✅ 使用 Cloudflare CDN 加速
✅ 更改DNS服务器至cloudflare

自定义域名

  1. 域名注册
    进入腾讯云官网，登录后找到域名注册然后去找自己心仪的域名购买就好，注意要提前实名才可以购买。
  2. GitHub pages绑定自定义域名
    现在可以到GitHub去绑定自己的域名，像下面这样。

    选择保存后可能会有下面的提示：

    这是因为还没有进行域名解析，所以我们可以回到腾讯域名管理那里，去进行域名的解析，从而通过域名正常访问对应的服务器。
  3. 域名解析
    进入腾讯云控制台，在这个界面里可以选择你的域名，DNS解析>我的域名，然后选择快速添加解析，如下图所示：


    选择网站解析>域名映射，然后填写你的GitHub pages的默认域名(用户名.github.io)，最后点击确认就可以自动完成解析了。

完成以上的步骤后，就已经可以通过你自己的域名去访问你的静态网站了，不过此时还只能使用http进行访问，还不能够通过https进行访问，我们在完成cloudflare的CDN加速的步骤中就可以完成https的设置。

cloudflare进行CDN加速

  1. 注册cloudflare账号
    在cloudflare官网进行账号注册或登录
    然后就会马上出现一个输入你自己域名的界面，输入完成域名后就会看到一个选择计划的界面，选择免费版本。
  2. 域名解析
    然后会到下面这样的界面，会让你添加DNS记录：

总结

这次域名解析花了我不少时间，一边看教程一边做的，虽说不是太难，但主要第一次做还是不太熟练。总结一下我的网站的发布过程吧，首先是使用hexo主题搭建网站然后部署到github上，然后又买了自己的域名，再想着让人们从搜索引擎上可以看到我的博客，我又登录必应站长平台发布了我的网站（为什么不是百度或者其他，原因是必应比较简单并且是我一直在用的。）所以又搞了网站地图这些乱七八糟的。总归来说，还是坚持下来了。
